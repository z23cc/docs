<Info>
  **前提条件**：请在继续之前安装 Node.js（19 版或更高版本）。<br />
  请在继续之前升级到 ```docs.json``` 并删除旧版 ```mint.json``` 文件。
</Info>

按照以下步骤在您的操作系统上安装和运行 Mintlify：

**步骤 1**：安装 Mintlify：

<CodeGroup>

  ```bash npm
  npm i -g mintlify
  ```

```bash yarn
yarn global add mintlify
```

</CodeGroup>

**步骤 2**：导航到文档目录（`docs.json` 文件所在位置）并执行以下命令：

```bash
mintlify dev
```

您的文档的本地预览将在 `http://localhost:3000` 上可用。

### 自定义端口

默认情况下，Mintlify 使用端口 3000。您可以使用 `--port` 标志自定义 Mintlify 运行的端口。例如，要在端口 3333 上运行 Mintlify，请使用此命令：

```bash
mintlify dev --port 3333
```

如果您尝试在已经使用的端口上运行 Mintlify，它将使用下一个可用端口：

```md
端口 3000 已在使用中。正在尝试使用 3001 代替。
```

## Mintlify 版本

请注意，每个 CLI 发布都与特定版本的 Mintlify 相关联。如果您的本地网站与生产版本不一致，请更新 CLI：

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```

```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## 验证链接

CLI 可以帮助验证文档中的引用链接。要识别任何损坏的链接，请使用以下命令：

```bash
mintlify broken-links
```

我们建议在您的 IDE 上使用扩展来识别和格式化 MDX。如果您是 VSCode 用户，请考虑使用 [MDX VSCode 扩展](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) 进行语法高亮显示，以及 [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) 进行代码格式化。

## 故障排除

<AccordionGroup>
  <Accordion title='错误：无法使用 darwin-arm64 运行时加载 "sharp" 模块'>

    这可能是由于 node 版本过旧所致。请尝试以下操作：
    1. 删除当前安装的 mintlify 版本：`npm remove -g mintlify`
    2. 升级到 Node v19 或更高版本。
    3. 重新安装 mintlify：`npm install -g mintlify`
  </Accordion>

  <Accordion title="问题：遇到未知错误">
  
    解决方案：转到设备根目录并删除 \~/.mintlify 文件夹。之后，再次运行 `mintlify dev`。
  </Accordion>
</AccordionGroup>

想了解 CLI 版本中的变化？[查看 CLI 更新日志。](https://www.npmjs.com/package/mintlify?activeTab=versions)
